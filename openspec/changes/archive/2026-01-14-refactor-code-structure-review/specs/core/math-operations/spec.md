## ADDED Requirements

### Requirement: 运算类型扩展机制
(SHALL) 系统应提供可扩展的题目生成机制，支持在不修改核心代码的情况下添加新的运算类型。

#### Scenario: 注册新运算类型
- **GIVEN** 开发人员需要添加新的运算类型（如幂运算）
- **WHEN** 创建对应的题目生成策略类
- **THEN** 可通过扩展机制注册到题目生成器中

#### Scenario: 使用扩展的运算类型生成题目
- **GIVEN** 已注册的新运算类型
- **WHEN** 调用题目生成器生成该类型题目
- **THEN** 返回符合新运算类型逻辑的题目

### Requirement: 题目生成策略接口
(SHALL) 系统应提供标准化的题目生成策略接口，确保所有运算类型实现一致的行为。

#### Scenario: 实现新的生成策略
- **GIVEN** 开发人员需要实现新运算类型的题目生成逻辑
- **WHEN** 实现IProblemGenerationStrategy接口
- **THEN** 新策略可无缝集成到题目生成器中

## MODIFIED Requirements

### Requirement: 数学题目生成
(SHALL) 系统应能生成五种数学运算类型的题目：加法、减法、乘法、有余数除法、无余数除法。生成的题目应符合配置的数值范围限制。系统应采用可扩展的策略模式实现题目生成。

#### Scenario: 生成加法题目
- **GIVEN** 整数位数配置为 2 位
- **WHEN** 生成加法题目
- **THEN** 返回两个 10-99 的随机数作为加数，正确结果为两数之和

#### Scenario: 生成减法题目（不允许负数结果）
- **GIVEN** 整数位数配置为 2 位，AllowNegativeResult = false
- **WHEN** 生成减法题目
- **THEN** 返回被减数和减数，确保被减数 ≥ 减数，结果为非负数

#### Scenario: 生成乘法题目
- **GIVEN** 整数位数配置为 2 位
- **WHEN** 生成乘法题目
- **THEN** 返回两个 10-99 的随机数作为乘数，正确结果为两数之积

#### Scenario: 生成有余数除法题目
- **GIVEN** 整数位数配置为 3 位
- **WHEN** 生成有余数除法题目
- **THEN** 返回被除数、除数、商和余数，满足：被除数 = 除数 × 商 + 余数，且 0 ≤ 余数 < 除数

#### Scenario: 生成无余数除法题目
- **GIVEN** 整数位数配置为 3 位
- **WHEN** 生成无余数除法题目
- **THEN** 返回被除数、除数、商，满足：被除数 = 除数 × 商，余数为 0
