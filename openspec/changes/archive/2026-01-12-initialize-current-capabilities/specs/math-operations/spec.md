## ADDED Requirements

### Requirement: 数学题目生成
系统应能生成五种数学运算类型的题目：加法、减法、乘法、有余数除法、无余数除法。生成的题目应符合配置的数值范围限制。

#### Scenario: 生成加法题目
- **GIVEN** 整数位数配置为 2 位
- **WHEN** 生成加法题目
- **THEN** 返回两个 10-99 的随机数作为加数，正确结果为两数之和

#### Scenario: 生成减法题目（不允许负数结果）
- **GIVEN** 整数位数配置为 2 位，AllowNegativeResult = false
- **WHEN** 生成减法题目
- **THEN** 返回被减数和减数，确保被减数 ≥ 减数，结果为非负数

#### Scenario: 生成乘法题目
- **GIVEN** 整数位数配置为 2 位
- **WHEN** 生成乘法题目
- **THEN** 返回两个 10-99 的随机数作为乘数，正确结果为两数之积

#### Scenario: 生成有余数除法题目
- **GIVEN** 整数位数配置为 3 位
- **WHEN** 生成有余数除法题目
- **THEN** 返回被除数、除数、商和余数，满足：被除数 = 除数 × 商 + 余数，且 0 ≤ 余数 < 除数

#### Scenario: 生成无余数除法题目
- **GIVEN** 整数位数配置为 3 位
- **WHEN** 生成无余数除法题目
- **THEN** 返回被除数、除数、商，满足：被除数 = 除数 × 商，余数为 0

### Requirement: 答案验证
系统应能验证用户提交的答案是否正确，支持普通运算和有余数除法的验证。

#### Scenario: 验证加法答案正确
- **GIVEN** 题目：25 + 37 = 62
- **WHEN** 用户提交答案 62
- **THEN** 返回验证通过（true）

#### Scenario: 验证加法答案错误
- **GIVEN** 题目：25 + 37 = 62
- **WHEN** 用户提交答案 60
- **THEN** 返回验证失败（false），提供正确答案 62

#### Scenario: 验证有余数除法答案正确
- **GIVEN** 题目：25 ÷ 7 = 3 余 4
- **WHEN** 用户提交商 = 3，余数 = 4
- **THEN** 返回验证通过（true）

#### Scenario: 验证有余数除法答案错误（商正确，余数错误）
- **GIVEN** 题目：25 ÷ 7 = 3 余 4
- **WHEN** 用户提交商 = 3，余数 = 3
- **THEN** 返回验证失败（false），提供正确答案 商 = 3，余数 = 4

#### Scenario: 验证有余数除法答案错误（商和余数都错误）
- **GIVEN** 题目：25 ÷ 7 = 3 余 4
- **WHEN** 用户提交商 = 4，余数 = 3
- **THEN** 返回验证失败（false），提供正确答案
