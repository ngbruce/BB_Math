## MODIFIED Requirements

### Requirement: 数学题目生成
(SHALL) 系统应能生成五种数学运算类型的题目：加法、减法、乘法、有余数除法、无余数除法。生成的题目应符合配置的数值范围限制。系统应根据选择的难度级别 (Easy/Medium/Hard) 调整题目参数,包括数值范围和可用的运算类型。难度定义如下:
- Easy (100以内加减法): 操作数和得数都是100以内
- Medium (100以内加减乘除): 操作数和得数都是100以内
- Hard (100以上加减乘除): 操作数和得数至少有一个在100以上

#### Scenario: 生成加法题目
- **GIVEN** 整数位数配置为 2 位
- **WHEN** 生成加法题目
- **THEN** 返回两个 10-99 的随机数作为加数，正确结果为两数之和

#### Scenario: 生成减法题目（不允许负数结果）
- **GIVEN** 整数位数配置为 2 位，AllowNegativeResult = false
- **WHEN** 生成减法题目
- **THEN** 返回被减数和减数，确保被减数 ≥ 减数，结果为非负数

#### Scenario: 生成乘法题目
- **GIVEN** 整数位数配置为 2 位
- **WHEN** 生成乘法题目
- **THEN** 返回两个 10-99 的随机数作为乘数，正确结果为两数之积

#### Scenario: 生成有余数除法题目
- **GIVEN** 整数位数配置为 3 位
- **WHEN** 生成有余数除法题目
- **THEN** 返回被除数、除数、商和余数，满足：被除数 = 除数 × 商 + 余数，且 0 ≤ 余数 < 除数

#### Scenario: 生成无余数除法题目
- **GIVEN** 整数位数配置为 3 位
- **WHEN** 生成无余数除法题目
- **THEN** 返回被除数、除数、商，满足：被除数 = 除数 × 商，余数为 0

#### Scenario: 根据 Easy 难度生成加减法题目
- **GIVEN** 当前难度为 Easy
- **WHEN** 生成加法或减法题目
- **THEN** 操作数应在10-99范围,且得数必须<100;如果得数>=100则重新生成

#### Scenario: 根据 Medium 难度生成四则运算题目
- **GIVEN** 当前难度为 Medium
- **WHEN** 生成加减乘除题目
- **THEN** 操作数应在10-99范围,且得数必须<100;如果得数>=100则重新生成

#### Scenario: 根据 Hard 难度生成四则运算题目
- **GIVEN** 当前难度为 Hard
- **WHEN** 生成加减乘除题目
- **THEN** 至少有一个操作数或得数>=100;如果所有数都<100则重新生成

#### Scenario: Hard 难度除法题目保证被除数>=100
- **GIVEN** 当前难度为 Hard
- **WHEN** 生成除法题目(无余数或有余数)
- **THEN** 被除数应>=100;如果被除数<100则重新生成
