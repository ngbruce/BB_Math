# 测试说明

## 项目类型

本项目使用的是**传统的 .NET Framework 4.6.2**，而不是 .NET Core 或 .NET 5+。

## 测试运行方法

### 在 Visual Studio 2019 中运行（推荐）

1. 在 Visual Studio 2019 中打开 `BB_Math.sln`
2. 点击菜单：**测试** → **测试资源管理器**
3. 点击 **运行全部** 按钮
4. 在测试资源管理器中查看测试结果

## 查看测试结果

在 Visual Studio 的测试资源管理器中可以看到：
- 通过的测试数量（绿色 ✓）
- 失败的测试数量（红色 ✗）
- 每个测试的详细信息

## 测试覆盖

项目包含完整的单元测试，目前共有 219 个测试用例，全部通过。

### 测试分类
- 常量类测试（AppConstantsTests）
- 异常处理测试（ExceptionHandlingTests）
- 配置管理测试
- 数学运算测试
- 日志系统测试
- 文件服务测试

## 常见问题

### Q: 为什么不升级到 .NET Core/.NET 5+?

A: 本项目主要作为学习项目，使用传统的 Windows Forms 和 .NET Framework 更适合当前的架构和需求。未来的版本可能会考虑迁移。

### Q: 测试失败怎么办？

A:
1. 在测试资源管理器中点击失败的测试
2. 查看错误消息和堆栈跟踪
3. 检查相关代码实现
4. 双击测试跳转到测试代码
5. 使用 Visual Studio 调试功能进行调试

### Q: 如何只运行特定的测试？

A:
1. 在测试资源管理器中找到要运行的测试
2. 右键点击测试
3. 选择"运行选定的测试"

### Q: 如何调试测试？

A:
1. 在测试资源管理器中右键点击要调试的测试
2. 选择"调试选定的测试"
3. 在测试代码中设置断点
4. 开始调试
